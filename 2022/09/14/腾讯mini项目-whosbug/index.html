腾讯mini项目-whosbug

Created2022-09-14|Updated2022-09-15
|Post View:

前言

暑假参加了学校与腾讯合作的mini项目，由于当时看见了 whosbug 项目是官方标明的 P1 难度，就想着难得参加腾讯的项目，而且使用的语言是之前就感兴趣的 Golang ，于是就选择了 whosbug

在项目过程中，也结识了给力的队友，给力的老师，给力的导师，从这表面14天，实则一个月的项目中，也确实接触了很多之前一点都不懂的东西，涉及了很多全新的领域，故写一篇博客来记录下

whosbug 概述

whosbug 分为对应的 CI 插件部分和 web 部分，其实说白了，就是一个用来让你甩锅甩的更准确，更有理有据的大程序

站在用户的角度看 whosbug，在你的项目发生报错的后，把对应的报错信息，如第三方的提交项目，出错的版本，以及发生错误的崩溃堆栈，提交到暴露给用户的 web 端，然后用户其实就等着出结果就可以了，在用户的视角里，这个提交完之后，web 端就会返回一个字符串，即这次报错的责任人，也就是造成这次错误最主要的人，还会对应返回一个小于等于1的浮点数，表示有多大的几率可以相信这次错误确确实实是这个人造成的，这样一次 whosbug 就完事了，即在用户角度，提交出错信息，返回责任人就ok

站在运行的角度，一份代码，或者说一个项目，要先进 CI 进行分析，如果以前没有分析过，那就拉取最新版本进行 antlr 语法分析，如果以前有分析过，那就看看要分析的这个版本是不是这个项目的最新版，是的话就不做操作，不是就拉取最新版，再进行分析，所有的这些分析结果，

前置学习

这个项目作为一门暑期的课程开设，需要面试，这也让我有点紧张，暑假的课程，在5月中就需要进行前置学习，并在学习期间做好笔记，对自己学习的东西，期间遇到的问题，对应进行的思考进行记录

当时要学习的有很多，我基本很多都是从头开始，比如 git 的各种操作，Golang 的基本语法，甚至还学了一点 gin 框架，当时还走了弯路，以为 antlr 是重点，为了看懂 antlr ，直接去现学了一天的 java，但事实上，最后只用到了 git 和 Golang

当时给的学习资料很多，就是去年的 whosbug 的 Github 仓库，当时也是不是很懂，把代码 download 下来之后乱看一通，觉得自己懂了，其实完全不懂

最后在端午假期的最后一天进行线上的笔试，我感觉我写的很烂，但是可能是我运气好，可能是导师比较喜欢我，最后我还是进了，虽然说有6个组，但是在小两百人里面，选出30人，我自己觉得还是比较幸运的，因为我是0基础，但是我认识的很多人其实都懂相关的东西，所以还是，感谢导师的捞人

项目进行

因为导师有点事情，7月末我们才开了第一次会，进行前置学习的进度对接，那时候好像其他组已经开始了一个周了，属于是起步就给爆了，但是后面大家加班也勤，最后其实留了大概4天的时间来测试，debug，排练等等收尾的东西，其实最后还是遵循了导师说的先紧后松的宗旨

由于第一天我就和导师交流了一下，抓了个 CI 插件在 Github 上的版本的虫，第一次会议导师就让我做会议纪要了，我是这样猜的，我也是第一次做这种多人项目，本来开会就挺紧张的，还要做记录，所以那次听的格外认真

那次会议基本就给项目定分工，定方向，定工期，效率还是很高的，不过在那次会之后，我确实没想到我之后会是整个项目的部署佬，也可以叫运维佬，顺便兼职当下测试佬
